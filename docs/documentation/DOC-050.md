# OpenLAG Self Lifecycle Documentation

## Repository Boundary Documentation

The OpenLAG repository root is the primary lifecycle boundary. Root `docs/*` is the authoritative source for version, change, feature, implementation, testing, and governance traceability.

`internal/dev-sandbox/docs` is intentionally secondary. It is used for compatibility and regression checks, not as primary release lifecycle evidence.

## Generated Evidence Policy

The repository intentionally versions generated evidence and runtime outputs (`logs/**`, `public/*.json`, freeze outputs captured for audits) for release reproducibility and auditability.

- Why versioned: to keep immutable audit trails for baseline/evolution validations and to expose drift as explicit Git diffs.
- Reproducibility expectation: generated outputs must be reproducible from source docs/contracts and controlled runtime conditions.
- Drift detection: mismatch between source artifacts and regenerated outputs is treated as lifecycle signal, not incidental noise.

## Tradeoff Statement

Versioning generated evidence improves deterministic audits and historical traceability, but increases repository size and merge friction. OpenLAG 0.5.0 accepts this tradeoff to prioritize lifecycle reviewability and compliance evidence continuity.

## Freeze Runtime Dependency Boundary

Freeze CLI behavior is global-command accessible, but HTML/PDF export reliability depends on project-local runtime dependencies and template availability in the active workspace. Deterministic export requires controlled execution context, dependency integrity, and consistent profile/contract inputs.
