# OpenLAG 0.5.0 Lifecycle Evolution

## Evolution Summary

VER-050 turns OpenLAG into an explicitly self-documented release lifecycle. The key difference from VER-040 is not only feature presence but traceability quality: 0.5.0 describes concrete changes, ownership, verification, and governance impact in first-class artifacts.

## Repository Boundary Change

The repository root is now the authoritative OpenLAG project boundary for lifecycle reporting. `docs/*` is the primary evidence source, while `internal/dev-sandbox/docs` is explicitly secondary and used for compatibility and regression checks.

## Branch And Workflow Change

- `main` remains the published 0.4.0 baseline reference.
- `develop` carries 0.5.0 lifecycle evolution and release-candidate validation.
- Freeze generation from root docs is treated as release evidence, not optional narrative output.

## Implementation Impact Areas

- CLI flows in `scripts/cli`: check/generate/freeze workflows tightened around lifecycle evidence generation.
- Core generation and freeze logic in `scripts/core`: deterministic transformation from source docs to runtime payloads and documentary exports.
- Contracts/profiles in `docs/contracts` and `profiles`: explicit governance and export behavior.

## Compatibility Notes

VER-050 is contract-compatible with baseline behavior while adding stronger relation density and lifecycle explainability. Teams can keep existing artifact types but should adopt richer relation linking to preserve migration clarity in generated documentation.
