# OpenLAG 0.4.0 Release Baseline

## Baseline Scope

VER-040 is the audited stable baseline before 0.5.0 lifecycle expansion. It provides valid contracts, relation payload generation, lint reporting, and portal metadata, but lifecycle traceability remains mostly implicit.

## Repository Boundary State

In this baseline, repository-root `docs/*` was still the observable OpenLAG project, but branch/version evolution details were not deeply modeled in source artifacts. Readers often needed branch comparison to understand exact deltas.

## Behavioral Baseline

- Contract loading and graph generation worked from root documentation.
- Validation checks (`check --profile`) enforced release and develop constraints.
- Generated outputs were deterministic for the same source state.

## Gap Closed By VER-050

What VER-040 lacks is explicit lifecycle navigation across change rationale, affected modules, migration implications, and governance evidence linkages. VER-050 addresses that gap by adding denser cross-artifact traceability.
