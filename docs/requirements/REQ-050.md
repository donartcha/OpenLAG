# Self-Documenting Lifecycle Boundary

## Lifecycle / Product Need
The platform requires a single, unambiguous source of truth for architecture and governance metadata. By restricting the primary lifecycle boundary to the repository root (`docs/` and `docs/contracts/`), we ensure that generated audits and freeze outputs are authentic and not contaminated by secondary experimental domains (such as the dev-sandbox).

## Rationale
In previous versions, there was ambiguity regarding which folders defined the authoritative contracts and models. Formalizing this boundary prevents configuration drift and split-authority bugs during releases.

## Constrained System Behavior
All command entrypoints (generate, check, build, freeze) must evaluate and validate the root context as the primary scope. Experimental environments must be isolated and verified separately.

## Acceptance Criteria
- `npm.cmd run check` succeeds when root artifacts comply with root contracts.
- Graph generation is restricted to the validated root-level documents.
- Any unauthorized boundary violations or mismatched contexts throw execution errors.
