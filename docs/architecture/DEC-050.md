# Repository Root As Primary Lifecycle Boundary

This decision represents the boundary change introduced in 0.5.0: lifecycle truth is produced from repository-root `docs/*` and `docs/contracts/*`.

It exists to avoid split authority between root and sandbox datasets. `internal/dev-sandbox` remains a compatibility and regression dataset, not the source of record for release lifecycle evidence.

The impact is architectural and operational: generation, check, and freeze workflows must be evaluated first in root context, then optionally in sandbox context.
