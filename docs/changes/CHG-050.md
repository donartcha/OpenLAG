# OpenLAG 0.5.0 Contract-Driven Evolution

## What Changed

OpenLAG 0.5.0 changes the lifecycle baseline from "project example inside docs only" to "repository root as first-class OpenLAG project." The lifecycle graph now has explicit root artifacts for version evolution, feature deltas, implementation footprint, validation evidence, and governance checks.

The freeze/export lifecycle adds version-scoped documentary output (`freeze --version`) and keeps deterministic output contracts across markdown/json/html/pdf.

## Where It Changed

- Lifecycle sources under `docs/versions`, `docs/changes`, `docs/features`, `docs/implementation`, `docs/testing`, `docs/documentation`, `docs/governance`.
- CLI and freeze orchestration under `scripts/cli` and `scripts/core/freeze.ts`.
- Contract and profile behavior under `docs/contracts` and `profiles/core/export-profiles`.
- Runtime graph payloads under `public/*.json` generated from root `docs/*`.

## Why It Changed

The 0.4.0 baseline was valid but too implicit for release audits: understanding exact behavior required reading branch diffs and code. 0.5.0 introduces explicit lifecycle artifacts so release reviewers can trace cause/effect from requirements to implementation and verification directly in the generated documentation.

## Behavioral And Workflow Impact

- Generation and preview workflows now treat repository root `docs/*` as the primary lifecycle dataset.
- `internal/dev-sandbox/*` remains secondary and is used for compatibility checks, not authoritative lifecycle reporting.
- Freeze documentation can now align with version snapshots instead of only profile filters.
- Release validation now ties check/profile/package/freeze outputs to explicit lifecycle artifacts.

## Migration Impact From 0.4.0

- Existing root `docs/*` data remains compatible, but 0.5.0 requires richer cross-artifact relations to expose traceability.
- Teams moving from 0.4.0 should add version-linked `CHANGE`, `FEATURE`, `CODE_ENTITY`, `TEST_CASE`, `DOCUMENTATION`, and `EVIDENCE` artifacts to avoid opaque freeze reports.
- Sandbox content is no longer a substitute for root lifecycle records.

## Compatibility And Determinism

0.5.0 preserves contract-driven compatibility: artifact/relation validation and profile behavior remain deterministic. Added lifecycle explicitness does not change graph semantics; it makes the same semantics inspectable without manual Git archaeology.
