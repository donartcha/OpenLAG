Why OpenLAG exists

Software changes continuously. The knowledge explaining why it exists, what it implements and what validates it usually decays. OpenLAG makes that lifecycle observable.

TraceabilityConnect intent to implementation

Requirements, features, code entities and tests become linked artifacts instead of disconnected documentation.

GovernanceMake gaps visible

Rules, checks, risks and evidence become part of the same architecture graph instead of separate audit notes.

FreezeGenerate deterministic dossiers

Export Markdown, HTML, JSON and PDF snapshots directly from the graph for reviews, releases and audit evidence.

The OpenLAG lifecycle

A repository-native workflow: human-readable Markdown, explicit YAML contracts, generated runtime payloads and a static portal.

Markdown + YAMLArchitecture artifacts and contract definitions live in Git.
Graph generationOpenLAG creates graph-data and runtime contract payloads.
ValidationLint profiles reveal broken relations and traceability gaps.
Impact analysisContract-defined propagation explains what a change may affect.
Freeze exportGenerate deterministic architecture documentation snapshots.

OpenLAG Lite

A low-friction starter path for teams that want to learn the core loop before adopting broader governance.

npx @donartcha/openlag init --starter
openlag check --profile develop
openlag freeze --profile starter --format markdown
